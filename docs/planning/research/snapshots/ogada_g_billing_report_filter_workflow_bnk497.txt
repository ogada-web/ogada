## ogada 4-ledger per-user 자동저장 (확인·git @77b1ea8/@fd0a3b3)

### DB (V170 + V171 integrity)
- table `billing_report_filters`
- UNIQUE(org_id, branch_id, user_id, year_month, variant)
- variants: charges | deposits | receipts | refunds

### BE API
- GET  /api/v1/billing/reports/filters?month=YYYY-MM&branchId=
- PUT  /api/v1/billing/reports/filters  { month, variant, q, depositPeriod, receiptBasis, branchId? }
- GET  /api/v1/billing/reports/{variant} → BillingReportFilterService.recordFromReportQuery() silent upsert (requireWriteScope=false @fd0a3b3)
- PUT explicit save → validateBranchWriteScope (HQ_ADMIN/BRANCH_ADMIN)

### FE wire (@77b1ea8 BillingReportPage.jsx + services.js)
- mount: fetchBillingReportFiltersApi({ month }) → hydrate search + depositPeriod + receiptBasis
- search submit / deposit tab / receipt tab → saveBillingReportFilterApi(payload) + fetchReport
- variants:
  · deposits: depositPeriod FULL | FIRST_HALF | SECOND_HALF
  · receipts: receiptBasis PAYMENT | CLAIM
  · charges/refunds: q search only

### FE tests (+3 cases BillingReportPage.test.jsx)
- persist depositPeriod on tab change → saveBillingReportFilterApi FIRST_HALF
- persist receiptBasis on tab change → saveBillingReportFilterApi CLAIM
- hydrate from saved filters → fetchBillingReportFiltersApi + fetchReport q/period

### BE tests (+1 @fd0a3b3)
- recordFromReportQueryShouldNotRequireWriteScope — autosave on GET report must not call validateBranchWriteScope
